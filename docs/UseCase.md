# Use Case Description

## Use Case: User Login

Actor: Participant, Judge, Admin

Description: Users log in to access the system.

Main Flow:
1. User enters email and password.
2. System checks account information.
3. System redirects user to the correct dashboard.

## Use Case: Submit Project

Actor: Participant

Description: A team submits their hackathon project.

Main Flow:
1. Participant logs in.
2. Participant selects their team.
3. Participant enters project information.
4. Participant submits project link.
5. System saves the submission.

## Use Case: Evaluate Project

Actor: Judge

Description: Judge evaluates submitted projects.

Main Flow:
1. Judge logs in.
2. Judge views project list.
3. Judge enters score and feedback.
4. System saves evaluation result.
