## Actors

- Participant
- Judge
- Admin

## Non-Functional Requirements

1. The system should be easy to use.
2. The system should protect user passwords.
3. The system should allow multiple users to access at the same time.
4. The system should store data correctly.
5. The system should be responsive on desktop and mobile.

## Use Case List

### Participant
- Register account
- Login
- Create team
- Submit project
- View result

### Judge
- Login
- View submitted projects
- Evaluate project
- Give feedback

### Admin
- Manage users
- Manage competition
- Manage teams
- Manage judges
- Publish results
